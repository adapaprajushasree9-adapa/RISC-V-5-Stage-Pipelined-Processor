# RISC-V 5-Stage Pipelined Processor on Basys3 FPGA

## Overview

This project presents the design and implementation of a **32-bit RISC-V (RV32I) 5-stage pipelined processor** developed in **Verilog HDL** and deployed on the **Basys3 FPGA (Xilinx Artix-7)** platform. The processor follows the classical pipelined CPU architecture, enabling multiple instructions to be executed concurrently across different stages of the pipeline, thereby improving overall throughput compared to a single-cycle processor.

To validate the functionality of the processor, a **Fibonacci sequence generation program** was implemented and executed on hardware. The successful execution of the program demonstrates correct instruction fetching, decoding, execution, memory access, and register write-back operations while maintaining proper pipeline synchronization and hazard resolution.

The design incorporates dedicated **pipeline registers**, **data forwarding mechanisms**, and a **hazard detection unit** to ensure correct execution of dependent instructions and minimize performance degradation caused by pipeline hazards.

A clock divider module was integrated to reduce the onboard 100 MHz Basys3 clock frequency, allowing easier observation and debugging of processor behavior during hardware testing.

The complete processor was simulated, synthesized, implemented, and verified using Xilinx Vivado before deployment onto the FPGA.

# Processor Architecture

The processor follows the conventional **5-stage RISC-V pipeline architecture**:

## 1. Instruction Fetch (IF)

* Fetches instructions from instruction memory.
* Updates the Program Counter (PC) to point to the next instruction.
* Supplies fetched instructions to the decode stage through the IF/ID pipeline register.

### Responsibilities

* Program Counter management
* Instruction memory access
* Sequential instruction fetching

---

## 2. Instruction Decode (ID)

* Decodes the fetched instruction.
* Generates control signals required for later stages.
* Reads source operands from the register file.
* Performs immediate value generation.

### Responsibilities

* Instruction decoding
* Register file read operations
* Immediate extraction
* Control signal generation

---

## 3. Execute (EX)

* Performs arithmetic and logical operations using the ALU.
* Calculates branch and jump target addresses.
* Evaluates branch conditions.
* Receives forwarded data when required to resolve hazards.

### Responsibilities

* ALU computation
* Address calculation
* Branch evaluation
* Operand forwarding

---

## 4. Memory Access (MEM)

* Executes load and store operations.
* Interfaces with data memory.
* Transfers memory results to the write-back stage.

### Responsibilities

* Data memory read
* Data memory write
* Memory access control

---

## 5. Write Back (WB)

* Writes computation or memory results back into the register file.
* Completes instruction execution.

### Responsibilities

* Register file update
* Final result write-back

---

# Pipeline Registers

To maintain synchronization between stages and enable concurrent instruction execution, dedicated pipeline registers are implemented:

* IF/ID Register
* ID/EX Register
* EX/MEM Register
* MEM/WB Register

These registers store intermediate data and control information generated by each stage, ensuring proper pipeline operation.

# Hazard Handling Mechanisms

Pipeline execution introduces hazards that can affect correctness. To address these challenges, the processor includes dedicated hazard management hardware.

## Data Forwarding Unit

The forwarding unit reduces unnecessary stalls by routing results directly from later pipeline stages back to the execute stage whenever possible.

### Benefits

* Reduces pipeline stalls
* Improves throughput
* Maintains correct execution of dependent instructions

---

## Hazard Detection Unit

Certain dependencies, particularly load-use hazards, cannot be resolved solely through forwarding. The hazard detection unit identifies such conditions and inserts stalls when necessary.

### Features

* Detects instruction dependencies
* Generates stall signals
* Prevents incorrect operand usage

---

## Control Hazard Management

The processor correctly handles branch-related control hazards by ensuring proper instruction flow and pipeline synchronization during branch evaluation.

# Core Components

The processor consists of the following RTL modules:

### Program Counter (PC)

Maintains the address of the current instruction being executed.

### Instruction Memory

Stores program instructions, including the Fibonacci sequence program.

### Register File

Provides storage for the 32 general-purpose RISC-V registers.

### Arithmetic Logic Unit (ALU)

Performs arithmetic and logical operations required by the instruction set.

### Control Unit

Generates control signals based on instruction decoding.

### Immediate Generator

Extracts and extends immediate values from instruction fields.

### Pipeline Registers

Enable stage-to-stage data transfer within the pipeline.

### Data Memory

Handles load and store operations.

### Forwarding Unit

Resolves data hazards through bypassing techniques.

### Hazard Detection Unit

Detects dependencies requiring pipeline stalls.

### Clock Divider

Reduces the FPGA clock frequency for hardware observation and debugging.
